## üîÑ Keeping Docs Updated

**When to update:**
- Core documents: After major milestones or architecture changes
- Feature docs: When features change significantly
- Test guides: When testing procedures change
- STATUS.md: After completing development phases

**What to avoid:**
- Creating new top-level docs without consolidating existing ones
- Duplicating information across documents
- Leaving outdated information

## üìù Contributing to Documentation

**Found an issue?**
- Missing information ‚Üí Add to appropriate core document
- Outdated info ‚Üí Update and verify
- Unclear explanation ‚Üí Rewrite for clarity

**Adding new features?**
- Update CURRENT-STATE.md with status
- Add API docs to DEVELOPER-GUIDE.md
- Create feature-specific doc if complex
- Update DESIGN.md if architecture changes

**Best practices:**
- Write for AI agents and humans
- Use clear headings and structure
- Include code examples
- Keep it concise but complete
- Use diagrams where helpful
