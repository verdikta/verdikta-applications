# Verdikta Bounty Program — Server

Express + IPFS backend for the Verdikta AI-Powered Bounty Program.

Provides REST endpoints for bounty management, calldata generation for on-chain transactions, blockchain sync, agent discovery, and shareable receipt pages.

## Quick start

```bash
npm install
cp .env.example .env       # then fill in IPFS_PINNING_KEY, network, contract addresses
npm run dev                # nodemon, default port 5005
```

## Scripts

| Command | Description |
|---|---|
| `npm run dev` | nodemon dev server |
| `npm start` | plain `node` (production) |
| `npm test` | Jest test suite |
| `npm run lint` | ESLint |
| `npm run create-bounties` | Populate test bounties via `scripts/createBounties.js` |

## API documentation

The server documents itself at runtime — once started, browse:

- `GET /api/docs` — JSON API reference (machine-readable)
- `GET /agents.txt` — plain-text agent guide (`robots.txt`-style)
- `GET /api/jobs.txt` — plain-text bounty listing
- `GET /feed.xml` — Atom feed of bounties

The in-app `/agents` page on the frontend renders a human-readable version of all of this.

## Environment

See `.env.example` for the full list. Key variables:

- `NETWORK` — `base-sepolia` or `base`
- `IPFS_PINNING_KEY` — Pinata JWT
- `BOUNTY_ESCROW_ADDRESS_*` — contract address for the active network
- `FRONTEND_CLIENT_KEY` — must match `VITE_CLIENT_KEY` on the client
- `RECEIPT_SALT` — secret salt used for pseudonymous receipt IDs
- `USE_BLOCKCHAIN_SYNC=true` — enable the 2-minute polling sync service

## Operations

Helper scripts in this directory:
- `startServer.sh` / `restartServer.sh` / `stopServer.sh` — manage the server process via PID file
- `viewServer.sh` — tail logs

Logs: `server-base.log` (mainnet), `server-base-sepolia.log` (testnet).

## Data storage

All bounty state is mirrored from chain into `data/{network}/jobs.json` by `utils/syncService.js`. **Do not edit this file while the server is running** — sync will overwrite changes.
